# Interview Problems

This section contains real interview-style problems categorized by difficulty level.

## Overview

These problems combine multiple concepts and require pattern recognition, optimization, and strong coding skills. They represent the types of questions you'll encounter in technical interviews.

| Difficulty | Count | Topics                                    | Expected Time | Status      |
| ---------- | ----- | ----------------------------------------- | -------------- | ----------- |
| Easy       | -     | Basic pattern application                 | 15-25 minutes  | Not Started |
| Medium     | -     | Combined patterns, moderate optimization  | 30-45 minutes  | Not Started |
| Hard       | -     | Multiple patterns, complex optimization   | 45-60 minutes  | Not Started |

## Difficulty Guidelines

### Easy ⭐
- Focus: Single pattern or basic data structure usage
- Skills: Pattern recognition, clean implementation
- Time: Should solve in 15-25 minutes
- Examples: Simple array manipulation, basic tree traversal
- **Topics to expect**:
  - Array basics (search, manipulation)
  - String basics (reversal, checking)
  - Basic linked list operations
  - Simple tree traversals
  - Hash table lookups

### Medium ⭐⭐
- Focus: Multiple patterns, moderate optimization
- Skills: Pattern combination, trade-off analysis
- Time: Should solve in 30-45 minutes
- Examples: Complex array problems, tree modifications
- **Topics to expect**:
  - Two pointers + sorting
  - Sliding window + hash table
  - Tree/graph traversal + modification
  - Binary search + custom comparators
  - Dynamic programming + arrays

### Hard ⭐⭐⭐
- Focus: Complex multi-pattern problems, optimal solutions
- Skills: Deep optimization, creative approach
- Time: Should solve in 45-60 minutes
- Examples: Building data structures, complex graph problems
- **Topics to expect**:
  - Multiple pattern combinations
  - Advanced dynamic programming
  - Graph algorithms (Dijkstra, etc.)
  - Segment trees, tries, advanced data structures
  - Complex backtracking with constraints

## Interview Strategy

### Before Coding (5-10 minutes)
1. **Clarify Requirements**
   - Ask clarifying questions
   - Confirm input/output format
   - Discuss edge cases

2. **Discuss Approach**
   - Outline your strategy (without coding)
   - Mention time and space complexity
   - Ask for feedback

3. **Start Simple**
   - Brute force is okay initially
   - Show you understand the problem
   - Then optimize

### While Coding (20-40 minutes)
1. **Write Clean Code**
   - Use meaningful variable names
   - Add comments for complex logic
   - Follow Swift conventions

2. **Test as You Go**
   - Test with provided examples
   - Consider edge cases
   - Walk through logic

3. **Optimize if Time**
   - Reduce time complexity
   - Reduce space complexity
   - Refactor for readability

### After Coding (5-10 minutes)
1. **Test Thoroughly**
   - Empty inputs
   - Single elements
   - Maximum sizes
   - Boundary conditions

2. **Discuss Trade-offs**
   - Why this approach?
   - Could you do better?
   - What are the constraints?

3. **Ask for Feedback**
   - Is there anything to improve?
   - Would you approach it differently?
   - Questions about the solution?

## Problem Solving Checklist

- [ ] Understand the problem completely
- [ ] Identify applicable patterns
- [ ] Outline approach before coding
- [ ] Write clean, readable code
- [ ] Test with examples
- [ ] Test with edge cases
- [ ] Analyze time/space complexity
- [ ] Optimize if time permits
- [ ] Explain your solution clearly

## Practice Guidelines

### For Easy Problems
- **Goal**: Solve in 15 minutes
- **Focus**: Speed + accuracy
- Aim for 100% success rate

### For Medium Problems
- **Goal**: Solve in 35-40 minutes
- **Focus**: Pattern recognition + optimization
- Aim for 80%+ success rate

### For Hard Problems
- **Goal**: Solve in 55-60 minutes
- **Focus**: Complex optimization + creativity
- Aim for 60%+ success rate

## Progression Path

1. **Build Confidence** - Master easy problems completely
2. **Recognize Patterns** - Identify patterns quickly
3. **Combine Patterns** - Solve medium problems effectively
4. **Optimize Deeply** - Tackle hard problems with confidence
